# 공지 관리 (Notice Management)

## 1. 개요
- **화면명**: 공지 관리 (NoticesPage)
- **목적**: 외부 CMS를 통해 등록된 공지사항 및 운영 가이드를 조회합니다.
- **특징**: 이 화면은 **조회 전용(Read-only)**이며, 공지글의 등록 및 수정은 외부 CMS 도구에서 이루어집니다.
- **주요 기능**: 공지글 목록 조회, 대상 파트너사별 필터링.

## 2. 기능 명세

### 2.1 공지 목록 (Notice List)
API를 통해 가져온 게시글 목록을 표시합니다.

- **데이터 소스**: `notices` (Mock Data / External API)
- **행 식별자**: `id`

#### 검색 및 필터
| 구분 | 필드명 | 타입 | 설명 |
| :--- | :--- | :--- | :--- |
| **필터** | 대상 파트너 | Select | `targetPartner` 필드 기준 필터링 (전체, A협력사, B협력사 등) |

#### 컬럼 정의
| 헤더명 | 매핑 데이터 | 비고 |
| :--- | :--- | :--- |
| **No** | `id` | |
| **제목** | `title` | |
| **대상 파트너** | `targetPartner` | 공지가 노출되는 파트너사 범위 |
| **대상 지역** | `targetRegion` | 공지가 노출되는 지역 범위 |
| **작성자** | `author` | |
| **작성일** | `createdAt` | |

### 2.2 연동 방식 (Integration)
- **구조**: 별도의 백엔드 서비스 또는 CMS(Content Management System) API를 호출하여 데이터를 조회합니다.
- **데이터 흐름**:
  1. 외부 CMS에서 관리자가 게시글 등록.
  2. 어드민 접속 시 API를 통해 노출 대상(파트너, 지역)에 맞는 공지 목록 Fetch.
  3. 리스트 렌더링.
