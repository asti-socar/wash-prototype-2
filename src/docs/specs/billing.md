# 청구 관리 (Billing Management)

## 1. 개요
- **화면명**: 청구 관리 (BillingPage)
- **목적**: 파트너사별 세차 수행에 대한 청구 내역을 조회하고, 정산의 기준이 되는 단가 정책을 관리합니다.
- **주요 기능**: 기간별 청구 이력 조회, 청구 상세 확인, 업체별/미션별 단가 정책 설정.

## 2. 기능 명세

### 2.1 청구 내역 (Billing History)
기본 뷰(`view="list"`)이며, 발생한 청구 건들을 리스트 형태로 제공합니다.

- **데이터 소스**: `billingData` (Mock Data)
- **행 식별자**: `id`

#### 검색 및 필터
| 구분 | 필드명 | 타입 | 설명 |
| :--- | :--- | :--- | :--- |
| **기간** | 기간 조회 | DatePicker | `period` State를 통해 기준일자 선택 (현재는 단일 날짜 필터로 구현) |

#### 컬럼 정의
| 헤더명 | 매핑 데이터 | 비고 |
| :--- | :--- | :--- |
| **청구 ID** | `id` | |
| **오더 ID** | `orderId` | |
| **파트너사** | `partner` | |
| **금액** | `amount` | 천 단위 콤마 포맷 (`toLocaleString`) |
| **상태** | `status` | `청구완료`(ok), `대기`(default) 등 Badge 표시 |
| **청구일** | `date` | |

#### 청구 상세 (Detail Drawer)
- **진입 트리거**: 리스트 행 클릭.
- **제공 정보**: 청구 ID, 오더 ID, 파트너사, 금액, 상태, 청구일 등 상세 정보 조회 (Read-only).

### 2.2 단가 정책 (Pricing Policy)
상단 '단가 정책' 탭 클릭 시 전환되는 뷰(`view="policy"`)입니다.

- **데이터 소스**: `policyData` (Mock Data)
- **행 식별자**: `id`

#### 컬럼 정의
| 헤더명 | 매핑 데이터 | 비고 |
| :--- | :--- | :--- |
| **파트너사** | `partner` | |
| **기본 단가** | `baseAmount` | 기본 세차 수행 단가 |
| **미션 단가** | `missionAmount` | 특수 미션 수행 시 추가 단가 |
| **관리** | `action` | 수정 버튼(`Edit` 아이콘) 제공 |

#### 정책 등록/수정 (Policy Drawer)
*현재 프로토타입에서는 UI 버튼만 제공되며, 실제 로직은 구현 예정입니다.*
- **기능**: 파트너사별 계약된 기본 단가 및 미션 수행 단가를 설정합니다.
- **입력 필드**: 파트너사명(Read-only/Select), 기본 단가(Number), 미션 단가(Number).
