# 지역 정책 관리 (Region Policy)

## 1. 목적
- 대한민국 시도(Region 1) 및 시군구(Region 2) 단위의 공통 세차 정책을 설정하고 관리합니다.
- 특정 구역(Zone)에 개별 정책이 설정되지 않은 경우, 해당 지역 정책이 기본값으로 적용됩니다.

## 2. 주요 관리 필드

| 필드명 (En) | 필드명 (Ko) | 데이터 타입 | 설명 | 비고 |
|:---|:---|:---|:---|:---|
| `Region1ID` | 지역1 ID | `String` | 시도 단위 고유 ID | 예: `R1-01` |
| `Region2ID` | 지역2 ID | `String` | 시군구 단위 고유 ID | 예: `R2-001` |
| `시도` | 시도 명칭 | `String` | 광역자치단체 이름 | 예: `서울`, `경기` |
| `시군구` | 시군구 명칭 | `String` | 기초자치단체 이름 | 예: `강남구`, `수원시` |
| `주기세차(일)` | 주기세차 경과일 | `Number` | 세차 오더가 발행되는 기준 일수 | |
| `라이트세차` | 라이트세차 여부 | `Enum` | 라이트 세차 발행 유무 | `Y`, `N` |
| `우천취소` | 우천취소 여부 | `Enum` | 우천 시 오더 자동 취소 여부 | `Y`, `N` |

## 3. 기능 요구사항

### 3.1. 화면 구성 및 데이터 로드
- **탭 구조**: "지역1 정책 (시도)"와 "지역2 정책 (시군구)" 두 개의 탭으로 구성합니다.
- **데이터 소스**: 
  - 지역1: `src/mocks/region1policy.json`
  - 지역2: `src/mocks/region2policy.json`
- **컴포넌트**: 프로젝트 공통 UI(`src/components/ui.jsx`)의 `Tabs`, `DataTable`, `Badge`, `Input`, `Select` 등을 사용합니다.

### 3.2. 지역1 정책 (시도) 탭
- **목록 조회**: 시도별 정책 리스트를 출력합니다.
- **수정 모드 전환**: 
  - 우측 상단 [수정] 버튼 클릭 시 테이블이 '편집 모드'로 전환됩니다.
  - 편집 모드에서는 `주기세차(일)`은 `Input`, `라이트세차`와 `우천취소`는 `Select` 박스로 변경됩니다.
- **저장 및 취소**:
  - [저장] 클릭 시 변경된 상태를 `editedData`에 반영하고 편집 모드를 종료합니다.
  - [취소] 클릭 시 변경 사항을 파기하고 초기 상태로 돌아옵니다.

### 3.3. 지역2 정책 (시군구) 탭
- **필터 및 검색**:
  - **시도 필터**: `Select` 컴포넌트를 사용하여 특정 시도를 선택하면 해당 지역의 시군구만 필터링합니다.
  - **시군구 검색**: `Input` 컴포넌트를 사용하여 시군구 이름을 검색합니다. **콤마(,)나 공백**으로 구분하여 여러 시군구를 동시에 검색할 수 있어야 합니다.
- **수정 모드**:
  - 지역1과 동일한 인라인 수정 기능을 제공합니다.
  - **필터 연동**: 특정 지역이 필터링되거나 검색된 상태에서도 해당 데이터들만 골라 수정하고 저장할 수 있어야 하며, 화면에 보이지 않는 데이터가 유실되지 않도록 관리합니다.
- **페이지네이션**: 데이터가 많으므로 한 페이지에 10개 혹은 20개씩 끊어서 보여줍니다.

### 3.4. UI/UX 규칙
- **Badge 활용**: 
  - `Y`인 경우: `tone="ok"` (녹색 계열)
  - `N`인 경우: `tone="default"` (회색 계열)
- **일관성**: 기존 관리자 페이지와 동일한 여백, 폰트, 컬러 시스템을 유지합니다.

## 4. 개발 참고사항 (코드 규칙)
- `DataTable` 사용 시 `accessorKey` 대신 **`key`** 속성을 사용합니다.
- 커스텀 셀 렌더링은 **`render: (row) => ( ... )`** 형식을 따릅니다.
- `Tabs` 컴포넌트는 `useState`를 통해 **`currentValue`**와 **`onClick`** 속성으로 상태를 직접 제어합니다.
- 모든 입력 컴포넌트는 `src/components/ui.jsx`에서 임포트하여 사용합니다.
