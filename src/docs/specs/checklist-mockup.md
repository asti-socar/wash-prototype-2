# 세차 관리 서비스 기능 명세서

## 1. 문서 개요

### 1.1 목적
본 문서는 세차 관리 서비스의 업무 프로세스, 기능 요구사항, 데이터 모델 및 시스템 연동 규칙을 정의한다.

### 1.2 범위
- 세차 발행 유형 분류 및 처리 흐름
- 세차 전/후 사진 촬영 프로세스
- 점검 항목 입력 및 유효성 검증
- 외부 시스템 연동 규칙

### 1.3 용어 정의

| 용어 | 설명 |
|------|------|
| 오더 | 세차 작업 요청 단위 |
| 입고 파트너 | 차량을 입고받아 세차를 수행하는 협력사 |
| 현장 파트너 | 고객 위치에서 세차를 수행하는 협력사 |
| 특수 세차 | 시트 탈거, 부품 교체 등 심화 세차 작업 |
| 협의 세차 | 고객과 협의하여 범위를 결정하는 세차 |
| 라이트 세차 | 간소화된 외관 중심의 세차 |
| 옥스트라 | 점검이력 및 장애관리를 담당하는 외부 시스템 |
| 사고카드 | 외관 파손 발생 시 연동되는 사고 접수 시스템 |

| 안전삼각대 | 버튼 | 정상 / 분실 | 분실 시 필수 |

## 6. 유효성 검증 규칙

### 6.1 사진 촬영 유효성

#### 6.1.1 조건부 사진 필수 규칙
점검 값이 '정상'이 아닌 경우(이상/분실/파손/위치선택 등) 해당 항목에 대한 사진 첨부가 필수로 활성화된다.

| 조건 | 필수 사진 |
|------|-----------|
| 유리창 = 이상 | 필수 |
| 배터리전압 = 노랑 또는 빨강 | 필수 |
| 안전삼각대 = 분실 | 필수 |
| 차량용 소화기 = 분실 | 필수 |
| 워셔액통 = 파손 | 필수 |
| 발판매트 = 분실 | 필수 |
| 타이어파스/펑크 ≠ 정상 | 필수 |
| 시동불가 = 이상 | 필수 |
| 본넷 = 고장 또는 파손 | 필수 |
| 타이어마모 ≠ 정상 | 필수 |
| 방향제 = 분실 | 필수 |
| 와이퍼 = 이상 | 필수 |
| 시트/폴딩 = 이상 | 필수 |
| 차량 경고등 선택 시 | 필수 |
| 외관 파손 ≠ 이상없음 | 필수 |

#### 6.1.2 사진 수량 제약
- 최소: 1장 (조건부 필수 항목)
- 최대: 10장 (모든 사진 입력 필드)

#### 6.1.3 유효성 검증 실패 시 동작
- 시나리오: 작업자가 '타이어 펑크'를 선택했으나 사진을 찍지 않고 제출 시도
- 대응: "이상 항목에 대한 증빙 사진이 필요합니다 (최소 1장)" 알림 팝업 노출 및 제출 차단

### 6.2 데이터 입력 유효성

| 필드 | 유효성 규칙 |
|------|-------------|
| 적산거리 | 숫자만 입력 가능, 최대 999,999 |
| 타이어트레드 | X.X 형식 (정수 1자리 + 소수점 1자리) |
| 시트 탈거 | 숫자 1자리 (0-9) |
| 부품교체 금액 | 숫자만 입력 가능, 최대 10자리 |
| 기타 코멘트 | 최대 100자 |

### 6.3 필수 항목 검증

#### 6.3.1 유형별 필수 항목 체크
- A~C유형: 공통 필수 12개 항목 + 유형별 특화 항목
- D유형(라이트): 라이트 전용 점검 항목

#### 6.3.2 제출 조건
모든 필수 항목 입력 완료 + 조건부 사진 등록 완료 시에만 데이터 전송 가능

## 7. 외부 시스템 연동

### 7.1 옥스트라 점검이력 연동

#### 7.1.1 연동 조건
- 점검 항목이 입력된 경우 모든 항목을 '옥스트라 점검이력'에 전송

#### 7.1.2 전송 데이터
- 오더 정보 (오더ID, 점검유형, 세차 유형 등)
- 차량 정보 (차량 번호, 차종 등)
- 점검 결과 (모든 점검 항목의 입력값)
- 사진 데이터 (세차 전/후 사진, 점검 증빙 사진)
- 작업자 정보
- 작업 일시

### 7.2 옥스트라 장애관리 연동

#### 7.2.1 연동 조건
- 점검 결과 중 '정상'이 아닌 항목이 존재하는 경우

#### 7.2.2 전송 데이터
- 오더 정보
- 차량 정보
- 이상 항목 목록 및 상세 내용
- 증빙 사진
- 긴급 조치 내용 (해당되는 경우)

### 7.3 사고카드 연동

#### 7.3.1 연동 조건
- '외관 파손' 항목에서 '이상없음' 외의 값을 선택한 경우
- D유형의 '외관 이상 유형'에서 '파손(운행 불가)'를 선택한 경우

#### 7.3.2 전송 데이터
- 오더 정보
- 차량 정보
- 파손 위치 (정면/운전석측면/조수석측면/후면/유리/기타)
- 파손 정도 (경미/파손)
- 증빙 사진

### 7.4 오더 재발행 연동

#### 7.4.1 연동 조건
- D유형(라이트 세차)의 '후속 조치' 선택 시

#### 7.4.2 재발행 규칙

| 후속 조치 선택값 | 재발행 발행 유형 |
|------------------|------------------|
| 완료 | 재발행 없음 |
| 외부세차 필요 | 외부 세차 오더 자동 생성 |
| 내부세차 필요 | 내부 세차 오더 자동 생성 |
| 내외부 세차 필요 | 내외부 세차 오더 자동 생성 |
| 입고 | 입고 오더 자동 생성 |

## 8. 워크플로우

### 8.1 전체 프로세스 흐름

```
[오더 접수] 
    ↓
[세차 유형 확인]
    ↓
[세차 전 사진 촬영] ← 세차 유형별 촬영 부위 상이
    ↓
[세차 작업 수행]
    ↓
[세차 후 사진 촬영] ← 세차 유형별 촬영 부위 상이
    ↓
[점검 항목 입력] ← 점검유형별 항목 상이
    ↓
[유효성 검증]
    ├─ 실패 → 오류 안내 및 수정 요청
    └─ 성공 ↓
[데이터 전송]
    ├─ 옥스트라 점검이력
    ├─ 옥스트라 장애관리 (이상 시)
    ├─ 사고카드 (외관 파손 시)
    └─ 오더 재발행 (라이트 후속조치 시)
    ↓
[작업 완료]
```

### 8.2 세차 유형별 사진 촬영 흐름

#### 8.2.1 내외부 세차
```
[세차 전] 외부 4장 + 내부 2~3장 (총 6~7장)
    ↓
[세차 후] 외부 14장 + 내부 12장 (총 26장)
```

#### 8.2.2 외부 세차
```
[세차 전] 외부 4장
    ↓
[세차 후] 외부 13장
```

#### 8.2.3 내부 세차
```
[세차 전] 내부 2~3장
    ↓
[세차 후] 외부 5장 + 내부 12장 (총 17장)
```

#### 8.2.4 라이트 세차
```
[세차 전] 없음
    ↓
[세차 후] 외부 2장 + 내부 2~3장 (총 4~5장)
```

### 8.3 점검유형별 입력 흐름

    ↓

    ↓

    ↓

    ↓

    ↓

    ↓

    ↓
