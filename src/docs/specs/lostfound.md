# 분실물 관리 (Lost & Found Management)

## 1. 개요
- **화면명**: 분실물 관리 (LostFoundPage)
- **목적**: 세차 서비스 중 습득된 분실물의 등록, 보관, 반환 처리 과정을 관리합니다.
- **주요 기능**: 분실물 목록 조회, 신규 등록, 처리 상태 업데이트.

## 2. 기능 명세

### 2.1 분실물 목록 (Lost Item List)
습득된 분실물 현황을 리스트로 제공합니다.

- **데이터 소스**: `items` (State Array)
- **행 식별자**: `id`

#### 검색 및 필터
*처리 상태에 따른 필터링 로직이 정의되어 있습니다.*

| 구분 | 필드명 | 타입 | 설명 |
| :--- | :--- | :--- | :--- |
| **상태 필터** | 처리 상태 | Select | 다음 상태값에 따른 필터링 지원:<br>- `찾는 중`<br>- `보관중`<br>- `경찰서 인계`<br>- `택배 발송` |

#### 컬럼 정의
| 헤더명 | 매핑 데이터 | 비고 |
| :--- | :--- | :--- |
| **분실물 ID** | `id` | |
| **습득물** | `item` | |
| **차량번호** | `plate` | |
| **상태** | `status` | 상태별 Badge 색상 적용 (`보관중`: ok, `찾는 중`: warn, 그 외: default) |
| **습득일** | `foundAt` | |

### 2.2 분실물 상세 및 등록 (Detail & Registration Drawer)
목록 클릭 또는 '분실물 등록' 버튼 클릭 시 우측 Drawer가 활성화됩니다.

- **모드 구분**: `isRegistering` 상태값에 따라 '등록' 또는 '상세/수정' 모드로 UI가 전환됩니다.

#### 입력 필드
| 필드명 | 매핑 데이터 | 입력 타입 | 설명 |
| :--- | :--- | :--- | :--- |
| **습득물 명** | `item` | Input | 분실물 종류 및 특징 (예: 지갑, 차키) |
| **차량번호** | `plate` | Input | 습득된 차량 번호 |
| **처리 상태** | `status` | Select | `찾는 중`, `보관중`, `경찰서 인계`, `택배 발송` 중 선택 |
| **보관 장소** | `location` | Input | 보관함 위치 또는 인계 장소 |
| **사진** | - | Placeholder | 습득물 사진 업로드/미리보기 영역 |

#### 처리 동작 (Actions)
1. **신규 등록 (Register)**
   - **트리거**: 목록 상단 '분실물 등록' 버튼 > Drawer 내 '등록' 버튼.
   - **동작**: 입력된 정보를 바탕으로 새로운 분실물 데이터를 생성하고 목록에 추가합니다.
   - **완료**: 등록 완료 알림 후 Drawer 닫힘.

2. **결과 업데이트 (Update)**
   - **트리거**: 목록 행 클릭 > Drawer 내 '저장' 버튼.
   - **동작**: 변경된 상태(예: 보관중 -> 택배 발송)나 정보를 저장합니다.
   - **완료**: 수정 완료 알림 후 Drawer 닫힘.
